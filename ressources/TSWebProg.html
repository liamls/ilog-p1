COURS

<p>Ceci est un paragraphe</p>

Des balises peuvent faire référence à des styles CSS :

<style>p { font-family: Arial; }</style>
<link rel="stylesheet" type="text/css" href="css/main.css">

Des balises peuvent faire référence à du code JavaScript 

<script>console.log('Bonjour');</script>
<script src="js/main.js"></script>
<button onclick="console.log('Bonjour !')">OK</button>.

COURS

DocType

  • <!DOCTYPE html> est une référence à une DTD (définition de type de document) ;
  • c'est la première ligne d'une page HTML5 ;
  • la partie <head> de la page n'est pas visible ;
  • le titre est montré comme titre de la fenêtre du navigateur.

Balises

  • Une balise comporte en général une partie ouvrante <p> et une partie fermante </p> :
<p>Bonjour le monde !</p>
  • Entre ses deux parties, une balise peut contenir :
    • du texte ;
    • d'autres balises.
<p>Attention à la <strong>marche</strong> !</p>
  • certaines balises n'ont qu'une partie ouvrante :
<br>

  • NB br comme break pour force un retour à la ligne.

Attributs

  • La partie ouvrante ou unique d'une balise peut comporter un ou des attributs dépendants de la balise :
<img src="icon.png" alt="Logo">
  • src est le nom de l'attribut, icon.png est sa valeur ;
  • certains attributs comme id ou class peuvent apparaître dans toute balise ;
  • la valeur d'id doit être unique dans la page car elle identifie un élément de manière unique ;
  • la valeur de l'attribut class identifie l'ensemble des éléments qui la portent.

Lien hyper texte

  • Les liens cliquables sont la base du succès du Web ;
  • on peut ainsi se déplacer vers une page ;
    • du même site : <a href="autrepage.html">Cliquer ici</a> ;
    • d'un autre site <a href="http://www.autresite.com/index.html">Autre site</a> ;
  • Pour repérer un emplacement dans une page, on utilise la balise <a> autrement :
<a id="ici">C'est ici</a>
  • NB l'attribut name n'existe plus en html5.
  • On peut ainsi se déplacer aussi vers un emplacement :
    • de la même page <a href="#ici">Cliquer ici</a> ;
    • d'une autre page du même site <a href="autrepage.html#la_bas">Cliquer ici</a> ;
    • d'une page d'un autre site : <a href="http://www.autresite.com/index.html#la_bas">Autre site là bas</a>.

COURS CSS

Attribut style

  • L'attribut style d'une balise HTML applique un style sur une balise au moyen de propriétés CSS :
<p style="font-style: italic; font-weight: bold;">Bonjour !</p>
  • font-style: italic; est une propriété font-style est le nom, italic est la valeur ;
  • L'attribut style est approprié quand le style est spécifique à 1 élément HTML.

Une propriété peut aussi :

  • se présenter sous une forme plus concise (border: 2 px solid green;) ;
  • regrouper plusieurs propriétés (commençant par border-) ;
  • avoir une valeur composée : 2 px solid green; 

Règles CSS

div {
	padding: 10px;
	background-color: grey;
	border-width: 2px;
	border-color: green;
	border-style: solid;
}
  • {} définit un bloc de propriétés ;
  • div est le sélecteur qui détermine à quoi s'appliquent les propriétés du bloc ;
  • ici la règle donc les propriétés s'appliquent à tous les éléments div

Les règles CSS peuvent se trouver :

  • soit directement dans une balise style d'une page HTML ;
  • soit dans une feuille de styles externe importée dans une ou plusieurs pages au moyen de balises link.

Balise style

<style>
règles CSS
</style>

Les règles d'une balise style s'appliquent :

  • aux éléments qu'elles ciblent ;
  • uniquement dans la page HTML qui la contient.

Balise link

<link rel="stylesheet" href="css/main.css">

Depuis une page HTML :

  • une balise link charge une feuille de style (ici un fichier main.css situé dans un sous-dossier css par rapport à la page HTML ) ;
  • les feuilles de styles peuvent être multiples.

Dans une feuille de styles, on trouve directement les règles CSS.

Les règles des feuilles CSS et des balises styles peuvent toutes entrer en concurrence.

La priorité est donnée aux règles dont les sélecteurs sont les plus spécifiques.

Sélecteurs CSS

Les sélecteurs CSS expriment des conditions visant à sélectionner les éléments auxquels va être appliquée la règle :

  • p pour la condition "est une balise p" ;
  • id0 pour "a l'id id0" ;
  • .nb pour "appartient à la classe nb" ;
  • [href] pour "a un attribut href" ;
  • [href=#] pour "qui a un attribut href égal à #" ;

Composition de sélecteurs

  • div p : éléments <p> quelque part dans un élément <div> ;
  • div > p : éléments <p> fils direct d'un élément <div> ;
  • div + p : éléments <p> juste après un élément <div> ;
  • div ~ p : éléments <p> frères d'un élément <div> ;

Pseudo-classe

Un sélecteur peut être suffixé par une pseudo-classe introduite par le signe ":"

sélecteur:pseudo-classe { propriétés }

Le style ne s'applique que si une condition est remplie :

  • :link si le lien est non visité ;
  • :hover si l'élément est survolé ;
  • :disabled si l'élément est désactivé (comme une zone de saisie ou un bouton) ;
  • :not(:disabled) : négation d'une condition (si l'élément n'est pas disabled)

Tous les sélecteurs : https://www.w3schools.com/cssref/css_selectors.asp.

4. Scripting du DOM

4.1 Requêtage

COURS

Désignation des éléments du DOM

Dans le DOM HTML, l'objet document désigne la page HTML.

Les méthodes querySelector et querySelectorAll de l'objet document ont un paramètre chaîne utilisant la syntaxe des sélecteurs CSS :

  • querySelector fournit un objet de type Element (ou null si non trouvé) réifiant le premier élément sélectionné ;
  • querySelectorAll fournit une NodeList d'objets réifiant les Elements sélectionnés ;

Ces méthodes sont également présentes pour tout objet réifiant un élément potentiellement parent.

  • Le résultat de  querySelector est un élément héritant de Element ou null si non trouvé.
  • Le résultat de querySelectorAll est une NodeList d'Elements

Pour en savoir plus, voir sur MDN Localisation des éléments DOM avec les sélecteurs.

NB :

  • Pour TypeScript, le type résultat de querySelector() est Element | null ;
  • or, le type Element n'est pas type compatible avec Element | null ;
  • le compilateur l'indiquera sur une telle ligne par exemple :

const e: Element = document.querySelector(".hello")

Si dans le contexte, on sait que l'élément sera trouvé, on peut dire au compilateur de ne rien signaler en suffixant la ligne par un point d'exclamation :

const e: Element = document.querySelector(".hello") !

4.2 Instrumentation

Instrumenter l'élément signifie qu'on le prépare à traiter un événement qui le concerne. Le traitement sera réalisé par une fonction "callback" appelée listener (elle écoute les événements).

De manière générale, un programmeur écrit une fonction callback non pour l'appeler mais pour qu'elle soit appelée par un environnement, ici, le navigateur.

COURS EventTarget

lnterfaces DOM

Les objets qui réifient les éléments HTML héritent pour la plupart de HTMLElement mais ce n'est pas le sommet de la hiérarchie d'héritage.

Dans le DOM, un élément concret hérite successivement de HTMLElement, Element, Node et EventTarget.

HTMLElement

Tout objet de ce type possède une propriété id qui fournit la valeur de l'attribut id de l'élément (ici bouton1 ou bouton2).

EventTarget

Tout objet EventTarget possède les méthodes suivantes :
  1. addEventListener(type: string, listener, options?) : s'abonner à la réception d'un type événements, quand il se produira la fonction listener sera appelée avec un Event en argument
    • type: string : type d'événement comme "click", "keyup", "mouseeenter"/"mouseleave",
    • listener :
      • soit une function(event: Event): void ;
      • doit un objet EventListener ;
    • options.capture: boolean : true => le listener reçoit l'événement avant toute target sous-jacente dans l'arbre DOM ;
    • options.once: boolean : true => le listener est désabonné après réception de l'événement ;
    • options.passive: boolean : true => evt.preventDefault() sera inopérant ;
  2. removeEventListener(type, listener) : se désabonner pour le type d'événements et le listener utilisés lors de l'abonnement ;
  3. dispatchEvent(event: Event) : déclencher un événement pour provoquer l'appel des listeners (en général, c'est le navigateur qui appelle cette méthode, mais on pourrait aussi l'appeler pour tester un listener par exemple)

Le listener peut être une fonction :

  • function(event: Event) {... }
  • (event: Event) => {...}

ou un objet implémentant l'interface EventListener qui spécifie l'existence de la méthode :

  • handleEvent(event: Event) "méthode listener"

EventListener

  • handleEvent(event: Event) "méthode listener"

Event

  • target: objet ayant déclenché l'événement (pas forcément le même que l'objet du addEventListener car les événements se propagent)
  • type: string est le type d'événement ;
  • eventPhase : number :
    • Event.CAPTURING_PHASE = 1 : phase de propagattion descendante (de window vers la cible) ;
    • Event.AT_TARGET = 2 : sur la cible ;
    • Event.BUBBLING_PHASE = 3 : phase de propagattion montante (de la cible vers window)
  • preventDefault() : empêcher le comportement par défaut :
    • envoi d'un formulaire si après contrôle, il se trouve être mal rempli ;
    • retour à la ligne sur touche Enter pour l'utiliser comme validation ;
  • stopPropagation() : stoppe ou empêche la phase montante.

NB

  • TypeScript détecte les faiblesses de typage du "framework navigateur"
  • e.g. un EventTarget n'est pas forcément un Element ;
  • on peut retyper en Element à l'occasion d'une extraction de constante :
    const target = event.target as Element

ou

    const target = <Element> event.target

Binding de fonction

En JavaScript, on peut affecter une méthode meth d'un objet o dans une fonction :

    const fn = o.meth

Cependant, fn perd toute référence à o :

  • Un appel fn() n'utilise pas o (le this utilisé dans meth ne désigne pas o) ;
  • o.fn() n'est pas possible car fn n'est pas une propriété de o.

Pour embarquer dans fn la référence à o, il faut la lier avec un objet o particulier :

    const fn = o.meth.bind(o)

Dès lors, fn() est équivalent à o.meth()

Propagation des événements

Les événements se propagent selon 2 phases :

  1. phase de capture : en descendant dans l'abre DOM (window vers cible) ;
  2. phase de bubbling : en remontant dans l'arbre DOM (cible vers DOM) ;
Un listener non cible utilisant l'option capture sera notifié avant les listeners sous-jacents.

4.4 Insertion dans le DOM

COURS Manipulation du DOM

Les méthodes et propriétés héritées de Element ou Node sont présentes dans tout Element ou Node.

Node

Les méthodes et propriétés héritées de Node offrent des possibilités de création, insertion, suppression, manipulation pour tout type de Node : Element, Text, Comment, DocumentType, Document, DocumentFragment...

propriétés readonly :

  • childNodes : NodeList contenant tous les Nodes fils, mise à jour en direct (dès qu'un Node est ajouté, retiré).
  • firstChild/lastChild : le premier/dernier Node fils direct ou null (pas de fils)
  • nextSibling : le prochain Node frère ou null (dernier)
  • nodeName : selon le noeud (nom de la balise, #text, #document...)
  • nodeType : (1=Element, 3=Text, 9=Document...)
  • nodeValue : valeur du Node
  • ownerDocument : le Document propriétaire ou null (absent du DOM)
  • parentNode : Node parent ou null (pas de parent)
  • parentElement : Element parent ou null (pas de parent ou parent non Element)
  • previousSibling : précédent Node frère immédiat ou null (premier)

propriété read/write :

  • textContent (exemple avec ce Node réifiant <div>a <p> b<b> c</b> d</p> e</div>)
    • read : fournit le texte contenu dans le Node et ses descendants : a b c d e
    • write : remplace tout le contenu par le nouveau texte (devient <div>nouveau texte</div>)

Méthodes

  • appendChild(n: Node) : insère comme dernier fils (détache n si ailleurs dans le DOM)
  • cloneNode(deep: boolean): Node : fournit une copie de lui-même (deep => lui + sous-arbre)
  • compareDocumentPosition(): number
  • contains(n: Node): boolean : true si n descendant de ce Node
  • getRootNode(): Node : retourne la racine de ce Node
  • hasChildNodes(): boolean : true si ce Node a un fils
  • insertBefore(n: Node, fils: Node): Node : insère n avant fils ; retourne n ou null (fils non fils de ce Node)
  • isEqualNode(n: Node): boolean : retourne true si n et ce Node sont "égaux" (l'égalité varie selon le type de Node mais ce n'est pas une comparaison d'arbres)
  • isSameNode(n: Node): boolean : retourne true si n et ce Node désignent le même Node
  • normalize() : nettoye tous les descendants Text (fusionne les adjacents, supprime les vides)
  • removeChild(n: Node) : supprime n si fils de ce Node, retourne n ou null (non fils)
  • replaceChild(n: Node, old: Node): Node : remplace old par n si old fils de ce Node ; retourne old ou null (non fils)

Element

Regroupe des caractéristiques communes à SVGElement et HTMLElement

  • création d'un Element (document joue donc le rôle de fabrique)
    • const elt: Element = document.createElement("div") ;
  • création d'un DocumentFragment
    • const frag: DocumentFragment = document.createDocumentFragment() ;

propriétés readonly

  • tagName: string = nom de l'élément (p, div, ...)
  • attributes: NamedNodeMap = collection d'Attr (qui hérite de Node)
  • classList: DOMTokenList = liste des classes décrit par l'attribut class (important pour le scripting des styles)
    • classList.contains(cls1: string): boolean ; true si contient la classe ;
    • classList.add(cls1: string, ...cls2: string) ; ajoute 1 ou ++ classes ;
    • classList.remove(cls1: string, ...cls2: string) : retire 1ou ++ classes ;
    • classList.replace(clsOld: string, clsNew: string) : remplace une classe par une autre ;
    • classList[i]: string : classe d'indice i ;
    • classList.toggle(cls1): boolean retire cls1 (si existe) ou l'ajoute (si n'existe pas) retourne true si existe après l'opération.
    • classList.forEach(fn) : itère avec une fonction fn(cls: string) ;
  • clientLeft : largeur de la bordure gauche de cet Element ;
  • clientTop : largeur de la bordure haute de cet Element ;
  • clientWidth : largeur interne de cet Element (padding+contenu) ;
  • scrollWidth : largeur interne du contenu de cet Element (padding+contenu incluant le débordement) ;
  • scrollHeight : hauteur interne du contenu de cet Element (padding+contenu incluant le débordement).

propriétés read/write

  • id : reflète l'attribut id ;
  • className : reflète attribut class ;
  • innerHTML : le contenu de cet Element ;
  • outerHTML : l'élément et son contenu ;
  • (webcomps) part : reflète l'attribut part (pour styler les éléments sous la shadow root ;
  • scrollLeft : décalage (> 0px) vers la gauche du contenu / bord gauche de cet Element ;
  • scrollTop : décalage (> 0 px) vers le haut du contenu / bord haut de cet Element ;

Méthodes

  • (webcomps) attachShadow(options): ShadowRoot : crée une shadow root, l'attache à cet element et la retourne ;
    • options.mode : "open" | "closed"
      • "open" : le contenu de la shadow root sera accessible au script externe ;
      • "closed" : ne sera pas accessible ;
    • options.delegatesFocus: boolean : true => la shadow root délèguera le focus à son premier élément pouvant le recevoir ;
  • getAttribute(namAttr: string): Object : retourne la valeur de l'attribut désigné par name et le retourne comme un Attr ;
  • getAttributeNames(): Array<string> : retourne un tableau des noms d'attributs.
  • getBoundingClientRect(): DOMRect : retourne le DOMRect (height/width + coordonnées ci-dessous / viewport en gris)

Coordonnées / Viewport (source MDN)

  • getClientRects(): Array<DOMRect> : retourne une collection de rectangles délimitant chaque CSS box de cet Element.
  • hasAttribute(namAttr: string): boolean : retourne true indiquant si l'élément a un attribut spécifié ou non.
  • insertAdjacentElement(pos: string, e: Element) : insère un Element à la position indiquée
  • insertAdjacentHTML(pos: string, html: string) : insère à la position indiquée le sous-arbre DOM décrit par html ;
  • insertAdjacentText(pos: string, txt: Text) : insère un Text à la position indiquée.
    • position: "beforebegin" | "afterbegin" | "beforeend" | "afterend"
    • "beforebegin" : insérer comme frère précédent ;
    • "afterbegin" : insérer comme comme premier fils ;
    • "beforeend" : insérer comme comme dernier fils ;
    • "afterend" : insérer comme prochain frère.

div.insertAdjacentHTML("beforeend", "<p> b<b> c</b> d</p>") insère avant </div> un Element contenant un Text, un Element et un Text.

  • querySelector(selectors: string): Element : recherche dans cet Element le premier Element satisfaisant à selectors ;
  • querySelectorAll(selectors: string): NodeList<Element> : recherche dans cet Element les Elements satisfaisant à selectors ;
    • selectors : sélecteurs séparés par ", " ;
  • releasePointerCapture : relâche (arrête) la capture de pointeur précédemment définie pour un événement pointeur spécifique.
  • removeAttribute() : supprime l'attribut nommé du noeud courant
  • setAttribute(namAttr: string, valAttr: string) : donne une valeur à un attribut de cet Element.
  • setPointerCapture() : fait capturer les futurs événements Pointer par cet Element.

5. Scripting CSS

COURS Manipulation des styles

On peut modifier les styles d'un élément par scripting en :

  • ajoutant / supprimant des styles à sa classList (méthodes contains, add, remove, replace, toggle) ;
  • tr.classList.toggle("showhide") étant défini le style : .showhide {display: none}
  • en ajoutant / modifiant / supprimant ses propriétés de style :
  • tr.style.display = "none"

Quand on manipule une propriété dont le nom comporte un tiret, utiliser le camel casing :

  • <td style="font-weight: bold;"> se scripte en td.style.fontWeight = "bold"
On peut modifier les styles d'un ensemble d'éléments en agissant sur les feuilles de styles (CSSStyleSheet) internes ou externes de la page :
  • const sts = document.styleSheets[2]

Pour les distinguer autrement que par leur indice, il peut être intéressant de leur donner un titre :

  • <style title="head">...</style>
  • if (sts.title == "head") { ... }

Une CSSStyleSheet comporte une CSSRuleList (liste de CSSRule) accessible comme un tableau :

  • const rules = sts.cssRules
  • const rule: any = rules[0]

On peut modifier une règle existante (propriétés selectorText et style)

  • rule.selectorText = "thead tr"
  • rule.style.setProperty("border", "1px solid grey") // ajout d'une propriété

On peut aussi ajouter de nouvelles règles :

  • sts.insertRule(`td i { color: blue;}`)

Annexe

COURS Manipulation des collections
  • entries retourne un itérateur sur les couples indice / valeur
  • every() true si une fonction est vraie pour tous les éléments du tableau,
  • filter retourne un tableau des éléments pour lesquels une fonction est vraie
  • find (ou findIndex) retourne l'élément (ou son indice) pour lequel une fonction est vraie
  • forEach applique une procédure aux éléments existants du tableau this.
  • Array.from crée un tableau des résultats de la fonction paramètre appliquée à chaque élément du tableau paramètre.
  • includes retourne true si l'élément paramètre est présent dans le tableau this.
  • indexOf retourne l'indice de la première occurrence de l'élément paramètre
  • join retourne les éléments d'un tableau joints dans une chaîne avec un séparateur
  • keys retourne un itérateur sur les indices du tableau
  • lastIndexOf retourne l'indice de la dernière occurrence de l'élément paramètre
  • map applique la fonction paramètre aux éléments du tableau this.
  • reduce applique une fonction d'accumulation aux éléments du tableau this.
  • reduceRight semblable à reduce avec un parcours de droite à gauche.
  • some retourne true si la fonction paramètre retourne true pour au moins un élément du tableau this.
  • sort : trie les éléments du tableau à l'aide d'une fonction de comparaison
